--- layout: post title: Application Express Views - Key discovery part 2 date: '2011-02-03T14:42:00.000-08:00' author: Kris Rice tags: modified_time: '2011-02-03T14:42:29.788-08:00' thumbnail: http://3.bp.blogspot.com/_HTPuwjEAZfk/TUspRUMzHCI/AAAAAAAAAYs/3B6igbQrYek/s72-c/change_name.png blogger_id: tag:blogger.com,1999:blog-37621059.post-5577168294678035990 blogger_orig_url: http://krisrice.blogspot.com/2011/02/application-express-views-key-discovery.html --- Yesterday, I showed how the new Foreign Key discovery works on a subset of dba views.  The only catch was that the end result had T_DBA_  prefixes for the names.  It's really easy to change that.

Once you import the views and convert them to tables, delete the views.  What you are left with is a bunch of tables prefixed with T_ .  Now right click on the relational model in the tree and choose Change Object Names Prefix.

This is a simple dialog, much like a search and replace.  Enter the old, the new, and what object types it applies to and everything is back to the base names you'd expect in a diagram.


The end result is a model of all the APEX_ views for 4.0.
